Hablemos de Git y GitHub

La organización debe ser muy importante para muchos de nosotros. Cuando creas archivos para tu proyecto, querrás mantenerlos en un único lugar y que se puedan compartir con los miembros del equipo, sin importar en qué parte del universo se encuentren. Con Git y GitHub, tendrás el rendimiento, seguridad y la flexibilidad en tur trabajos. Veamos de que trata cada uno.

  1. ¿Quién inventó el sistema de control de versión Git y por qué?

    Git es un proyecto de código abierto maduro y con un mantenimiento activo que desarrolló originalmente Linus Torvalds, el famoso creador del kernel del sistema operativo Linux, en 2005.
    Usar un sistema de control de versiones (VSC), permite revertir archivos y proyectos enteros a un estado anterior, comparar cambios a lo largo del tiempo, ver quién modificó por última vez algo que puede estar causando un problema, quién introdujo un error y cuándo, y mucho más.

  2. ¿A quién pertenece actualmente Github y por qué?

    El 4 de junio de 2018, Microsoft compró GitHub. Esto se debe a que GitHub tiene un valor estratégico mayor para Microsoft que el valor que representa a Amazon ó Google, por eso pueden pagar más.
    Microsoft tiene que competir con el líder indiscutible a día de hoy, la todopoderosa Amazon con AWS (servicios en la nube). Qué mejor forma de hacerlo que estar en contacto directo con los desarrolladores que trabajan en esos servicios, ver sus necesidades y ofrecerles herramientas que faciliten su integración con GitHub.

  3. ¿Hay otra forma que no sea la terminal para trabajar con Github?

    Si, Github Desktop.
    GitHub Desktop es una aplicación que te habilita para interactuar con GitHub utilizando una GUI en vez de la línea de comandos o de un buscador web.
